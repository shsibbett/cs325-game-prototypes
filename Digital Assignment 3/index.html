Misstep

  • Originality: What is original about your game?
    • The player must escape from a room without touching the walls.
    • The walls are invisible.
    • A red x is left behind wherever a player touches a wall.
    • The player leaves behind a white trail showing where they have been to help them get through the room.
  • Technical Merit: What is interesting about your game technology?
    • The trail that shows where the player has walked safely before is generated using bit map data.
  • Prototype Postmortem: What did you learn from this prototype? What was the easiest or hardest part of making it?
    • I learned how to change the visibility of sprites and how to have the player die on collision.
    • I also learned how to use bit maps to create a trail and how to create checkpoints for the player.
    • The easiest part was changing the visibility of the walls and to add red x's where the player died.
    • The hardest part was creating a tilemap that wasn't too hard for the player to find their way through.

    • Guy spritesheet from Curt - cjc83486
    • Red X image
    • Door sprite
    • Chunk sprite from the Ship Trail example
    • Splat sound effect
    • Door opening/closing sound effect
    • Tiled was used to create the tilemap.
    • Tileset I used for the unseen walls.
  • Prototype Closest Other Game: Which other game most closely resembles your game? If you are borrowing code from a Phaser Example, you must say so here. If you borrow code from elsewhere, you must say so here.
    • The code for player movement was taken from the Phaser Starstruck example.
    • I used code from the Phaser Ship Trail example to generate the dot trail.
    • I used both the Alpha Text and Chained Tweens examples from Phaser for the "Checkpoint reached!" text.
    • Code from the Sound Complete Phaser example was used for reloading the door sprite after the door opening/closing sound effect.
  • High Concept: A one-sentence summary of your game.
    • You have been captured by a murderous psychopath and must escape.
  • Theme: State which of the Storymatic themes you used.
    • I used the caught theme.
  • Mandated Variety: State which input, randomness, genre, and play style you used in this prototype.
    • The player uses the arrow keys to move.
    • There aren't really any random elements aside from how the player actions.
    • The game is a 2D, top-down side-scroller.
    • The play style is that the player must make mistakes in order to succeed and escape.
  • Prototype Goal: What game mechanic is this prototype evaluating?
    • This prototype is evaluating the mechanic that the player must die repeatedley in order to beat the game.
    • X's marking where the player has died, a trail of where they have been, and checkpoints were added to help the player find the exit.
  • Player Experience Goals: What experience do you want players to have when playing your game?
    • Overall, players should have fun and feel like they have been presented with a challenge.
    • Hopefully, players should not experience frustration as multiple elements have been added to help them escape the room.
